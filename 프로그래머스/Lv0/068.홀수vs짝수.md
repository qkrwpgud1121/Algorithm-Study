# 홀수 vs 짝수

## 문제 설명
정수 리스트 `num_list`가 주어집니다. 가장 첫 번째 원소를 1번 원소라고 할 때, 홀수 번째 원소들의 합과 짝수 번째 원소들의 합 중 큰 값을 return 하도록 solution 함수를 완성해주세요. 두 값이 같을 경우 그 값을 return합니다.

## 제한사항
- 5 ≤ `num_list`의 길이 ≤ 50
- -9 ≤ `num_list`의 원소 ≤ 9

## 입출력 예
| num_list | result |
| :-- | :-- |
| [4, 2, 6, 1, 7, 6] | 17 |
| [-1, 2, 5, 6, 3] | 8 |

## 입출력 예 설명
입출력 예 #1
- 홀수 번째 원소들의 합은 4 + 6 + 7 = 17, 짝수 번째 원소들의 합은 2 + 1 + 6 = 9 이므로 17을 return합니다.

입출력 예 #2
- 홀수 번째 원소들의 합은 -1 + 5 + 3 = 7, 짝수 번째 원소들의 합은 2 + 6 = 8 이므로 8을 return합니다.

## 배운점
- if-else나 삼항연산자를 사용하여 큰 값을 비교하는 대신 이전 문제에서 공부했던 Swift내장함수 `max`를 활용하는 방법을 복습했다.
- 삼항연산자는 조건에 따른 값을 반환하는 순수한 표현식으로만 사용되어야 하며 내부에서 외부 변수의 값을 변경하는 **사이드 이펙트**를 발생시키는 것은 피해야 하는 것을 알았다.
- Keep It Simple, Stupid 원칙
    - `reduce`와 같은 고차함수는 데이터를 변환하거나 도출하는 순수 함수 목적으로 사용해야하며 클로저 내부에서 외부 변수의 값을 조작하는것은 피해야 한다는것을 알았다.
    - 코드를 무조건 짧고 멋지게 짜는것보다 Keep It Simple, Stupid 원칙에 따라 누가 봐도 의도가 명확히 보이는 `for-in`, `if-else`가 유지보수성에서 좋을 때가 있다는것을 알게되었다.
- 고차 함수(선언형)와 제어문(명령형)의 기준 확립
    - 고차함수(`map`,`filter`,`reduce`등)는 외부 상태나 변수를 건드리지 않고 원본 데이터에서 내가 원하는 형태의 새로운 결과값을 꺼내 쓸때 사용하는것이 좋다.
    - 제어문(`for-in`,`if-else`)은 데이터를 순회하면서 외부 변수의 상태를 변경하거나 분기처리 흐름 제어 등 구체적인 행동을 지시할때 사용하는것이 가독성과 유지보수 면에서 좋다.
    - 즉 결과값을 반환할때는 고차함수를 상태를 변경해야할때는 제어문을 선택하는 기준을 정하게 되었다.

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ num_list:[Int]) -> Int {
    
    var sumEven = 0
    var sumOdd = 0
    
    return num_list.enumerated().reduce(into: 0) { (result, element) in
        
        if element.offset.isMultiple(of: 2) {
            sumEven += element.element
        } else {
            sumOdd += element.element
        }
        
        result = sumEven > sumOdd ? sumEven : sumOdd
    }
}
```

### 두번째 풀이
```swift
func solution(_ num_list:[Int]) -> Int {
    let sumEven = stride(from: num_list.startIndex, to: num_list.endIndex, by: 2).map { num_list[$0] }.reduce(0, +)
    let sumOdd = stride(from: num_list.startIndex + 1, to: num_list.endIndex, by: 2).map { num_list[$0] }.reduce(0, +)
        
    return max(sumEven, sumOdd)
}
```

### 세번재 풀이
```swift
func solution(_ num_list: [Int]) -> Int {
    var sumEven = 0
    var sumOdd = 0
    
    for (index, value) in num_list.enumerated() {
        if index.isMultiple(of: 2) {
            sumEven += value
        } else {
            sumOdd += value
        }
    }
    
    return max(sumEven, sumOdd)
}
```
