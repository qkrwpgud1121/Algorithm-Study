# 공백으로 구분하기 2

## 문제 설명
단어가 공백 한 개 이상으로 구분되어 있는 문자열 `my_string`이 매개변수로 주어질 때, `my_string`에 나온 단어를 앞에서부터 순서대로 담은 문자열 배열을 return 하는 solution 함수를 작성해 주세요.

## 제한사항
- `my_string`은 영소문자와 공백으로만 이루어져 있습니다.
- 1 ≤ `my_string`의 길이 ≤ 1,000
- `my_string`의 맨 앞과 맨 뒤에도 공백이 있을 수 있습니다.
- `my_string`에는 단어가 하나 이상 존재합니다.

## 입출력 예
| my_string | result |
| :-- | :-- |
| " i    love  you" | ["i", "love", "you"] |
| "    programmers  " | ["programmers"] |

## 입출력 예 설명
입출력 예 #1
- 예제 1번의 `my_string`은 " i    love  you"로 공백을 기준으로 단어를 나누면 "i", "love", "you" 3개의 단어가 있습니다. 따라서 ["i", "love", "you"]를 return 합니다.

입출력 예 #2
- 예제 2번의 `my_string`은 "    programmers  "로 단어는 "programmers" 하나만 있습니다. 따라서 ["programmers"]를 return 합니다.

## 이전 학습 내용 적용
- 해당 문제 역시 이전 문제와 마찬가지로 `split`을 사용하여 반환된 `Substring`을 `map`을 사용하여 타입을 변환하였다.
- 이전 문제에서 학습했던 `split` 직후 `map`을 사용하여 많은 연산을 수행하면 많은 메모리 복사가 발생하여 크래시가 발생할 수 있다.

- `split`과 `components`의 차이
    - 해당 문제 처럼 연속된 여러개의 공백이 주어지면 `components(separatedBy: " ")`를 사용하면 `["", "i", "", "love"]`와 같이 빈 문자열이 배열에 포함되는 문제가 발생한다.
    - 하지만 `split(separator:)`를 사용하면 메서드의 내부에서 `omittingEmptySubsequences: true`라는 파라미터가 기본값으로 설정되어있어 연속된 빈 문자열을 자동으로 제거 해주어 원하는 문자열만 반환을 받을 수 있다.

## 나의 풀이
```swift
func solution(_ my_string:String) -> [String] {
    my_string.split(separator: " ").map { String($0) }
}
```
