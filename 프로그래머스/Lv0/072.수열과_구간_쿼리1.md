# 수열과 구간 쿼리 1

## 문제 설명
정수 배열 `arr`와 2차원 정수 배열 `queries`이 주어집니다. `queries`의 원소는 각각 하나의 `query`를 나타내며, `[s, e]` 꼴입니다.

각 `query`마다 순서대로 `s` ≤ `i` ≤ `e`인 모든 `i`에 대해 `arr[i]`에 1을 더합니다.

위 규칙에 따라 `queries`를 처리한 이후의 `arr`를 return 하는 solution 함수를 완성해 주세요.

## 제한사항
- 1 ≤ `arr`의 길이 ≤ 1,000
    - 0 ≤ `arr`의 원소 ≤ 1,000,000
- 1 ≤ `queries`의 길이 ≤ 1,000
    - 0 ≤ `s` ≤ `e` < `arr`의 길이
    
## 입출력 예
| arr | queries | result |
| :-- | :-- | :-- |
| [0, 1, 2, 3, 4] | [[0, 1],[1, 2],[2, 3]] | [1, 3, 4, 4, 4] |

## 입출력 예 설명
입출력 예 #1
- 각 쿼리에 따라 `arr`가 다음과 같이 변합니다.

| i | queries[i] | arr |
| :-- | :-- | :-- |
| - | - | [0, 1, 2, 3, 4] |
| 0 | [0,1] | [1, 2, 2, 3, 4] |
| 1 | [1,2] | [1, 3, 3, 3, 4] |
| 2 | [2,3] | [1, 3, 4, 4, 4] |

- 따라서 [1, 3, 4, 4, 4]를 return 합니다.

## 배운점
- 첫번째 풀이에서 외부 변수를 선언해야 한다는 부담이 있지만 가독성 측면에서 매우 좋다고 생각을 했다.
- 두번째 풀이는 첫번재 풀이의 외부 변수를 없애보고자 고차함수를 사용해서 문제를 해결해 보았다.
- 간단한 연산에서는 고차함수를 고집하기 보다는 중첩 제어문을 선택하는것이 가독성면에서 더 좋다는것을 알게 되었다.

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ arr:[Int], _ queries:[[Int]]) -> [Int] {
    
    var numArr = arr
    
    for query in queries {
        for i in query[0]...query[1] {
            numArr[i] += 1
        }
    }
    
    return numArr
}
```

### 두번째 풀이
```swift
func solution(_ arr:[Int], _ queries:[[Int]]) -> [Int] {
    return queries.reduce(into: arr) { result, query in
        for i in query[0]...query[1] {
            result[i] += 1
        }
    }
}
```
