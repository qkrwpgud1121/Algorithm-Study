# 수열과 구간 쿼리 2

## 문제 설명
정수 배열 `arr`와 2차원 정수 배열 `queries`이 주어집니다. `queries`의 원소는 각각 하나의 `query`를 나타내며, `[s, e, k]` 꼴입니다.

각 `query`마다 순서대로 `s` ≤ `i` ≤ `e`인 모든 `i`에 대해 `k`보다 크면서 가장 작은 `arr[i]`를 찾습니다.

각 쿼리의 순서에 맞게 답을 저장한 배열을 반환하는 solution 함수를 완성해 주세요.
단, 특정 쿼리의 답이 존재하지 않으면 -1을 저장합니다.

## 제한사항
- 1 ≤ arr의 길이 ≤ 1,000
    - 0 ≤ arr의 원소 ≤ 1,000,000
- 1 ≤ queries의 길이 ≤ 1,000
    - 0 ≤ s ≤ e < arr의 길이
    - 0 ≤ k ≤ 1,000,000
    
## 입출력 예
| arr | queries | result |
| :-- | :-- | :-- |
| [0, 1, 2, 4, 3] | [[0, 4, 2],[0, 3, 2],[0, 2, 2]] | [3, 4, -1] |

## 입출력 예 설명
입출력 예 #1

- 첫 번째 쿼리의 범위에는 0, 1, 2, 4, 3이 있으며 이 중 2보다 크면서 가장 작은 값은 3입니다.
- 두 번째 쿼리의 범위에는 0, 1, 2, 4가 있으며 이 중 2보다 크면서 가장 작은 값은 4입니다.
- 세 번째 쿼리의 범위에는 0, 1, 2가 있으며 여기에는 2보다 큰 값이 없습니다.
- 따라서 [3, 4, -1]을 return 합니다.

## 나의 풀이
```swift
func solution(_ arr:[Int], _ queries:[[Int]]) -> [Int] {
    queries.map { query in
        let (s, e, k) = (query[0], query[1], query[2])
        return arr[s...e].filter { $0 > k }.min() ?? -1
    }
}
```

## 고차 함수 개념 다시 적립
- `map` (변환: Transform)
    - 배열의 각 요소를 하나씩 꺼내서 자신이 정한 규칙대로 변환하여 새로운 배열을 만든다.
    - 입력 개수와 출력 개수가 항상 일치한다.
    - **활용**: String 배열을 Int 배열로 변환
- `filter` (추출: Selection)
    - 각 요소중 자신이 정한 조건(Boolean)에 통과되는 요소만 골라서 남긴다.
    - 결과의 배열은 원래의 배열보다 개수가 적거나 같다.
    - **활용**: 검색, 제외하기, 짝수만 골라내기 등 특정 조건의 데이터만 보여줘야할때 사용
- `reduce` (응축: Accumulate)
    - 배열의 모든 요소를 하나로 합쳐서 단 하나의 결과값을 만든다.
    - 초기값이 무조건적으로 필요하며 루프를 돌때마다 기존값과 현재값을 합한다.
    - **활용**: 총액 계산, 여러 문자열 합치기, 상태값 누적 업데이트 등
- `compactMap` (안전한 변환: Safe Transform)
    - `map` 과 같이 변환하지만 결과가 nil인 요소들은 버리며 옵셔널값은 자동으로 푼다.
    - 실패할 간능성이 있는 변환에 사용한다.
    - **활용**: String 배열을 Int 배열로 변활할때 숫자가 아닌 요소는 제거 및 Optional 배열에서 nil을 뺄때 사용
- `flatMap` (평탄화: Flatten)
    - 중첩된 배열을 하나의 배열로 합해준다.
    - 2차원 배열을 1차원으로 만들때 유용하다.
    - **활용**: 2차원 배열 1차원으로 변환
