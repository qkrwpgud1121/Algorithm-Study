# 배열에서 문자열 대소문자 변환하기

## 문제 설명
문자열 배열 `strArr`가 주어집니다. 모든 원소가 알파벳으로만 이루어져 있을 때, 배열에서 홀수번째 인덱스의 문자열은 모든 문자를 대문자로, 짝수번째 인덱스의 문자열은 모든 문자를 소문자로 바꿔서 반환하는 solution 함수를 완성해 주세요.

## 제한사항
- 1 ≤ strArr ≤ 20
    - 1 ≤ strArr의 원소의 길이 ≤ 20
    - strArr의 원소는 알파벳으로 이루어진 문자열 입니다.

## 입출력 예

| strArr | result |
| :-- | :-- |
| ["AAA","BBB","CCC","DDD"] | ["aaa","BBB","ccc","DDD"] |
| ["aBc","AbC"] | ["abc","ABC"] |

## 입출력 예 설명

### 입출력 예 #1
- `strArr[0]`과 `strArr[2]`는 짝수번째 인덱스의 문자열이므로 모두 소문자로 바꿔서 "aaa"와 "ccc"가 됩니다.
- `strArr[1]`과 `strArr[3]`는 홀수번째 인덱스의 문자열인데 원래 대문자이므로 그대로 둡니다.
- 따라서 ["aaa","BBB","ccc","DDD"]를 return 합니다.

### 입출력 예 #2
- `strArr[0]`은 짝수번째 인덱스의 문자열이므로 소문자로 바꿔서 "abc"가 됩니다.
- `strArr[1]`은 홀수번째 인덱스의 문자열이므로 대문자로 바꿔서 "ABC"가 됩니다.
- 따라서 ["abc","ABC"]를 return 합니다.

## 배운점
- `indices`를 사용하여 배열의 인덱스 범위만 안전하게 추출할 수 있다는 것을 알게 되었다.
- `indices`를 사용하면 `0..<count`를 사용하는 왜 안전하며 `eumerated`처럼 튜플로 생성하는 오버해드가 없어 인덱스만 활용하거나 원본 배열에 직접적으로 접근할때 성능과 메모리 부분에서 좋다.

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ strArr:[String]) -> [String] {
    strArr.enumerated().map { $0.0.isMultiple(of: 2) ? $0.1.lowercased() : $0.1.uppercased() }
}
```

### 두번재 풀이
```swift
func solution(_ strArr:[String]) -> [String] {
    strArr.enumerated().map { index, value in
        index.isMultiple(of: 2) ? value.lowercased() : value.uppercased()
    }
}
```

### 세번째 풀이
```swift
func solution(_ strArr:[String]) -> [String] {
    strArr.indices.map { index in
        index.isMultiple(of: 2) ? strArr[index].lowercased() : strArr[index].uppercased()
    }
}
```

## 풀이 이유
- 첫번째 풀이에서는 `enumerated`를 통해 인덱스와 값을 추출하고 **클로저의 단축 인자와 튜플 요소 접근($0.0, $0.1)**을 사용하여 문제를 해결하였다.
- 두번째 풀이에서는 튜플 요소 접근을 사용했던 부분을 가독성을 위해 클로저 내부에 매개변수를 사용하였다.
- 세번째 풀이에서는 `enumerated`을 대신해 배열의 인덱스만을 가져오는 `indices`를 사용하여 튜플의 생성 비용을 제거하였다.
