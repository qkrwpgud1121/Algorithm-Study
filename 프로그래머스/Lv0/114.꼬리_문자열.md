# 꼬리 문자열

## 문제 설명
문자열들이 담긴 리스트가 주어졌을 때, 모든 문자열들을 순서대로 합친 문자열을 꼬리 문자열이라고 합니다. 꼬리 문자열을 만들 때 특정 문자열을 포함한 문자열은 제외시키려고 합니다. 예를 들어 문자열 리스트 ["abc", "def", "ghi"]가 있고 문자열 "ef"를 포함한 문자열은 제외하고 꼬리 문자열을 만들면 "abcghi"가 됩니다.

문자열 리스트 `str_list`와 제외하려는 문자열 `ex`가 주어질 때, `str_list`에서 `ex`를 포함한 문자열을 제외하고 만든 꼬리 문자열을 return하도록 solution 함수를 완성해주세요.

## 제한사항
- 2 ≤ `str_list`의 길이 ≤ 10
- 1 ≤ `str_list`의 원소의 길이 ≤ 10
- 1 ≤ `ex`의 길이 ≤ 5

## 입출력 예
| str_list | ex | result |
| :-- | :-- | :-- |
| ["abc", "def", "ghi"] | "ef" | "abcghi" |
| ["abc", "bbc", "cbc"] | "c" | "" |

## 입출력 예 설명
입출력 예 #1
- 본문과 동일합니다.

입출력 예 #2
- 리스트 안의 모든 문자열이 "c"를 포함하므로 빈 문자열을 return합니다.

## 배운점
- `filter`와 `contains`를 사용하여 `Array`이면서 내부 요소가 `String`이고 찾아야 하는 조건이 연속된 글자 패턴 일때 Swift 내장 검색 엔진을 사용하는 `contains`와 고차함수인 `filter`를 조합하는 것이 성능과 가독성에서 뛰어나다는것을 다시한번 느꼈다.
- 또한 `joined`를 사용하여 불필요한 반복문이나 임시 변수를 사용하지 않고 문자열 컬랙션을 시퀸스 상태에서 바로 이어붙여 메모리의 오버헤드를 막을 수 있었다.

## 나의 풀이
```swift
func solution(_ str_list:[String], _ ex:String) -> String {
    str_list.filter { !$0.contains(ex) }.joined()
}
```
