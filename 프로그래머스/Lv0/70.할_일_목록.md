# 할 일 목록

## 문제 설명
오늘 해야 할 일이 담긴 문자열 배열 `todo_list`와 각각의 일을 지금 마쳤는지를 나타내는 boolean 배열 `finished`가 매개변수로 주어질 때, `todo_list`에서 아직 마치지 못한 일들을 순서대로 담은 문자열 배열을 return 하는 solution 함수를 작성해 주세요.

## 제한사항
- 1 ≤ `todo_list`의 길이 1 ≤ 100
- 2 ≤ `todo_list`의 원소의 길이 ≤ 20
    - `todo_list`의 원소는 영소문자로만 이루어져 있습니다.
    - `todo_list`의 원소는 모두 서로 다릅니다.
- `finished[i]`는 true 또는 false이고 true는 `todo_list[i]`를 마쳤음을, false는 아직 마치지 못했음을 나타냅니다.
- 아직 마치지 못한 일이 적어도 하나 있습니다.

## 입출력 예
| todo_list  finished | result |
| :-- | :-- | :-- |
| ["problemsolving", "practiceguitar", "swim", "studygraph"] | [true, false, true, false]    ["practiceguitar", "studygraph"] |

## 입출력 예 설명
입출력 예 #1
- 예제 1번의 `todo_list` 중에서 "problemsolving"과 "swim"은 마쳤고, "practiceguitar"와 "studygraph"는 아직 마치지 못했으므로 `todo_list`에서 나온 순서대로 담은 문자열 배열 ["practiceguitar", "studygraph"]를 return 합니다.

## 배운점
- 두개의 배열을 병렬로 처리할때 `enumerated`를 사용하지 않고 `zip`을 사용하여 두 배열을 튜플로 묶는 방법을 알게 되었다.
- 두 배열의 길이가 달라도 짧은쪽에 맞춰 안전하게 종료되어 Index Out of Range 오류 발생을 방지할 수 있다.
- 하지만 세개의 배열을 순회하기 위해 `zip`을 중첩하여 사용하게 된다면 최악의 가독성을 지닌 코드가 탄생할수 있기 때문에 처음부터 연관된 데이터를 하나의 구조체(Struct) 객체로 생성했어야 한다.
- `$0`의 클로저 축약 인자와 `.0`튜플의 원소 접근자를 알게 되었으며 이를 통해 불필요한 매개 변수 네이밍을 생략하고 가독성이 좋은 선언형 데이터 파이프라인을 사용하수 있다.

- 배열에 요소를 추가시 `+=`를 사용하면 임시 배열을 메모리에 할당하게 되는 오버헤드가 발생한다, 해서 제어문 내부에서는 `append`를 사용하는것이 메모리에 좋다는것을 알게 되었다.

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ todo_list:[String], _ finished:[Bool]) -> [String] {

    finished.enumerated().reduce(into: []) { result, element in
        if !element.element {
            result += [todo_list[element.offset]]
        }
    }
}
```

### 두번째 풀이
```swift
func solution(_ todo_list:[String], _ finished:[Bool]) -> [String] {
    
    var result: [String] = []
    
    for (index, value) in finished.enumerated() {
        if !value {
            result += [todo_list[index]]
        }
    }
    
    return result
}
```

### 세번째 풀이
```swift
func solution(_ todo_list:[String], _ finished:[Bool]) -> [String] {
    zip(todo_list, finished).filter { !$0.1 }.map { $0.0 }
}
```
