# 배열 만들기 4

## 문제 설명
정수 배열 `arr`가 주어집니다. `arr`를 이용해 새로운 배열 `stk`를 만드려고 합니다.

변수 `i`를 만들어 초기값을 0으로 설정한 후 `i`가 `arr`의 길이보다 작으면 다음 작업을 반복합니다.
- 만약 `stk`가 빈 배열이라면 `arr[i]`를 `stk`에 추가하고 `i`에 1을 더합니다.
- `stk`에 원소가 있고, `stk`의 마지막 원소가 `arr[i]`보다 작으면 `arr[i]`를 `stk`의 뒤에 추가하고 `i`에 1을 더합니다.
- `stk`에 원소가 있는데 `stk`의 마지막 원소가 `arr[i]`보다 크거나 같으면 `stk`의 마지막 원소를 `stk`에서 제거합니다.
위 작업을 마친 후 만들어진 `stk`를 return 하는 solution 함수를 완성해 주세요.

## 제한사항
- 1 ≤ `arr`의 길이 ≤ 100,000
    - 1 ≤ `arr`의 원소 ≤ 100,000
    
## 입출력 예
| arr | result |
| :-- | :-- |
| [1, 4, 2, 5, 3] | [1, 2, 3] |

## 입출력 예 설명
입출력 예 #1
- 각 작업을 마친 후에 배열의 변화를 나타내면 다음 표와 같습니다.

| i | arr[i] | stk |
| :-- | :-- | :-- |
| 0 | 1 | [] |
| 1 | 4 | [1] |
| 2 | 2 | [1, 4] |
| 2 | 2 | [1] |
| 3 | 5 | [1, 2] |
| 4 | 3 | [1, 2, 5] |
| 4 | 3 | [1, 2] |
| - | - | [1, 2, 3] |

- 따라서 [1, 2, 3]을 return 합니다.

## 나의 풀이
```swift
func solution(_ arr:[Int]) -> [Int] {
    var stk: [Int] = []
    var i: Int = 0
    
    while i < arr.count {
        if stk.isEmpty {
            stk.append(arr[i])
            i += 1
        } else if stk.last! < arr[i] {
            stk.append(arr[i])
            i += 1
        } else if stk.last! >= arr[i] {
            stk.removeLast()
        }
    }
    return stk
}
```

## 배운점
- 단락 평가(Short-circuit)라는 것을 자세히 다시 공부하게 되면서 선행으로 확인하는 좌측 조건의 값이 false이면 우측의 조건을 확인하지 않는다는 것을 알게되었으며 이러한 방식으로 인해 빈 배열에 접근할때 발생하는 런타임 오류를 방지할 수 있다.	
- 이전 문제에서 공부했던 `reduce`관하여 좀더 깊이 공부하면서 `while`루프를 사용하여 직접적으로 `i`를 제어하는 대신 `reduce`를 사용하여 각 요소를 순차적으로 받아오며 요소를 어떻게 처리할지 정의 해주는 방법을 공부하게 되었다.

## 새로운 풀이
```swift
func solution(_ arr:[Int]) -> [Int] {
    return arr.reduce(into: []) { stk, element in
        while !stk.isEmpty && stk.last! >= element {
            stk.removeLast()
        }
        
        stk.append(element)
    }
}
```
