# l로 만들기

## 문제 설명
알파벳 소문자로 이루어진 문자열 `myString`이 주어집니다. 알파벳 순서에서 "l"보다 앞서는 모든 문자를 "l"로 바꾼 문자열을 return 하는 solution 함수를 완성해 주세요.

## 제한사항
- 1 ≤ `myString` ≤ 100,000
    - `myString`은 알파벳 소문자로 이루어진 문자열입니다.

## 입출력 예
| myString | result |
| :-- | :-- |
| "abcdevwxyz" | "lllllvwxyz" |
| "jjnnllkkmm" | "llnnllllmm" |

## 입출력 예 설명
입출력 예 #1
- 0 ~ 4번 인덱스의 문자 "a","b","c","d","e"는 각각 "l"보다 앞서는 문자입니다. 따라서 "l"로 고쳐줍니다.
- 그 외의 문자는 모두 "l"보다 앞서지 않는 문자입니다. 따라서 바꾸지 않습니다.
- 따라서 "lllllvwxyz"을 return 합니다.

입출력 예 #2
- 0번, 1번, 6번, 7번 인덱스의 문자 "j","j","k","k"는 각각 "l"보다 앞서는 문자입니다. 따라서 "l"로 고쳐줍니다.
- 그 외의 문자는 모두 "l"보다 앞서지 않는 문자입니다. 따라서 바꾸지 않습니다.
- 따라서 "llnnllllmm"을 return 합니다.

## 배운점
- "a=1", "b=2", ... 와 같이 별도의 Dictinary를 생성할 필요 없이 내부에서 문자를 숫자로 인식하는 것을 활용하여 `Character` 고유의 대소 비교 연산자를 활용해 구분 할수 있다는 것을 알게 되었다.
- 반환된 문자 배열 `[Character]`를 다시 하나의 문자열로 합치려면 `String`, `.joined()`를 사용하는 것보다 맨앞에 전체를 `String()` 초기화 생성자를 사용하는것이 메모리 오버헤드 없이 고속 압축하는 방법임을 알게 되었다.

## 나의 풀이
```swift
func solution(_ myString:String) -> String {
    String(myString.map { $0 < "l" ? "l" : $0 })
}
```
