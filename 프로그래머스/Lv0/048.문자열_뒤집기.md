# 문자열 뒤집기

## 문제 설명
문자열 `my_string`과 정수 `s`, `e`가 매개변수로 주어질 때, `my_string`에서 인덱스 `s`부터 인덱스 `e`까지를 뒤집은 문자열을 return 하는 solution 함수를 작성해 주세요.

## 제한사항
- `my_string`은 숫자와 알파벳으로만 이루어져 있습니다.
- 1 ≤ `my_string`의 길이 ≤ 1,000
- 0 ≤ `s` ≤ `e` < `my_string`의 길이

## 입출력 예
| my_string | s | e | result |
| :-- | :-- | :-- | :-- |
| "Progra21Sremm3" | 6 | 12 | "ProgrammerS123" |
| "Stanley1yelnatS" | 4 | 10 | "Stanley1yelnatS" |

## 입출력 예 설명
입출력 예 #1
- 예제 1번의 `my_string`에서 인덱스 6부터 인덱스 12까지를 뒤집은 문자열은 "ProgrammerS123"이므로 "ProgrammerS123"를 return 합니다.

입출력 예 #2
- 예제 2번의 `my_string`에서 인덱스 4부터 인덱스 10까지를 뒤집으면 원래 문자열과 같은 "Stanley1yelnatS"이므로 "Stanley1yelnatS"를 return 합니다.

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ my_string:String, _ s:Int, _ e:Int) -> String {
    
    var charArray = Array(my_string)
    charArray[s...e].reverse()
    return String(charArray)
}
```

### 두번째 풀이
```swift
func solution(_ my_string:String, _ s:Int, _ e:Int) -> String {
    
    let charArray = Array(my_string)
    return String(charArray[..<s] + charArray[s...e].reversed() + charArray[(e + 1)...])
}
```

## 배운점
- 언제나 고차함수가 좋은것이 아니다라는 것을 느낄수 있었던 문제였다.
- 첫번째 풀이처럼 변형이 가능한 배열을 만들고 `reverse` 사용하면 추가적인 메모리 할당 없이 원본 배열에서 요소들의 자리만 변경한다. 또한 범위가 직관적으로 확인하기 쉽다.
- 두번째 풀이처럼 고차함수를 사용하여 억지로 자르고 붙일때 해당하는 부분에대한 메모리를 할당을 해줘야 하여 성능면에서 떨어진다.
- `reverse`, `reversed`: 이 둘의 차이는 `reverse`는 원본 배열의 뒤집은 후에 아무것도 반환하지 않는다 반면 `reversed`는 뒤집힌 새로운 결과를 반환한다.
