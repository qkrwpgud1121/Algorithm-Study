# 문자열 묶기

## 문제 설명
문자열 배열 `strArr`이 주어집니다. `strArr`의 원소들을 길이가 같은 문자열들끼리 그룹으로 묶었을 때 가장 개수가 많은 그룹의 크기를 return 하는 solution 함수를 완성해 주세요.

## 제한사항
- 1 ≤ `strArr`의 길이 ≤ 100,000
    - 1 ≤ `strArr`의 원소의 길이 ≤ 30
    - `strArr`의 원소들은 알파벳 소문자로 이루어진 문자열입니다.

## 입출력 예

| strArr | result |
| :-- | :-- |
| ["a","bc","d","efg","hi"] | 2 |

## 입출력 예 설명

입출력 예 #1
- 각 문자열들을 길이에 맞게 그룹으로 묶으면 다음과 같습니다.

| 문자열 길이 | 문자열 목록 | 개수 |
| :-- | :-- | :-- |
| 1 | ["a","d"] | 2 |
| 2 | ["bc","hi"] | 2 |
| 3 | ["efg"] | 1 |

- 개수의 최댓값은 2이므로 2를 return 합니다.

## 배운점
- `reduce(into:)`를 사용할때 초기값의 타입을 `[]`, `[:]`, `Array<Type>()`등을 사용하여 명시적으로 타입을 지정할 필요가 있다는것을 다시한번 알게 되었다.
- `[]`, `[:]`와 같이 초기값을 전달 하더라도 컴파일러가 내부 로직을 분석하여 자동적으로 타입 추론을 한다는것을 알게되었다.
- 딕셔너리의 전체 요소를 클로저를 사용해 비교를 하는 대신 `.values`를 사용해 직접 접근하여 `.max()`를 사용하여 가독성을 높였다.

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ strArr:[String]) -> Int {
    
    let countArr = strArr.reduce(into: [:]) { counts, str in
        counts[str.count, default: 0] += 1
    }
    
    let maxCount = countArr.max(by: {$0.value < $1.value})
    
    return maxCount?.value ?? 0
}
```

### 두번째 풀이
```swift
func solution(_ strArr:[String]) -> Int {
    
    let countMap = strArr.reduce(into: [:]) { counts, str in
        counts[str.count, default: 0] += 1
    }
    
    return countMap.values.max() ?? 0
}
```
