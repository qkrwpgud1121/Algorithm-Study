# 9로 나눈 나머지

## 문제 설명
음이 아닌 정수를 9로 나눈 나머지는 그 정수의 각 자리 숫자의 합을 9로 나눈 나머지와 같은 것이 알려져 있습니다.
이 사실을 이용하여 음이 아닌 정수가 문자열 `number`로 주어질 때, 이 정수를 9로 나눈 나머지를 return 하는 solution 함수를 작성해주세요.

## 제한사항
- 1 ≤ `number`의 길이 ≤ 100,000
- `number`의 원소는 숫자로만 이루어져 있습니다.
- `number`는 정수 0이 아니라면 숫자 '0'으로 시작하지 않습니다.

## 입출력 예
| number | result |
| :-- | :-- |
| "123" | 6 |
| "78720646226947352489" | 2 |

## 입출력 예 설명
입출력 예 #1
- 예제 1번의 `number`는 123으로 각 자리 숫자의 합은 6입니다. 6을 9로 나눈 나머지는 6이고, 실제로 123 = 9 × 13 + 6입니다. 따라서 6을 return 합니다.

입출력 예 #2
- 예제 2번의 `number`는 78720646226947352489으로 각자리 숫자의 합은 101입니다. 101을 9로 나눈 나머지는 2이고, 실제로 78720646226947352489 = 9 × 8746738469660816943 + 2입니다. 따라서 2를 return 합니다.

## 나의 풀이
```swift
func solution(_ number:String) -> Int {
    number.map { Int(String($0))! }.reduce(0, +) % 9
}
```

## 배운점
- 문자열을 순회하거나 배열로 변환하면 각 요소의 형태가 `String`이 아닌 `Character` 라는것을 알았다.
- **`wholeNumberValue`**
    - Swift의 `Character`타입만이 가지고 있는 고유한 프로퍼티이다.
    - 숫자 형태의 문자를 정수형으로 반환한다. ex) "0" -> 0
    - 숫자가 아닌 문자는 안전하게 nil을 반환한다. ex) "A" -> nil
    - 옵셔널 정수를 반환 한다.
- **`compactMap`**:
    - 배열을 순회하며 nil인 값을 안전하게 제거하고 정상적인 값만 Unwrapping 하여 모아준다.
    - 해당문제에서 `wholeNumberValue`가 반환하는 옵셔널 값을 강제 추출(`!`)의 위험 없이 안전하게 정수의 배열을 얻을 수 있다.
    
## 새로운 풀이
```swift
func solution(_ number:String) -> Int {
    number.compactMap { $0.wholeNumberValue }.reduce(0, +) % 9
}
```
