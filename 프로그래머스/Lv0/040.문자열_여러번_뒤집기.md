# 문자열 여러 번 뒤집기

## 문제 설명
문자열 `my_string`과 이차원 정수 배열 `queries`가 매개변수로 주어집니다. `queries`의 원소는 [s, e] 형태로, `my_string`의 인덱스 s부터 인덱스 e까지를 뒤집으라는 의미입니다. `my_string`에 `queries`의 명령을 순서대로 처리한 후의 문자열을 return 하는 solution 함수를 작성해 주세요.

## 제한사항
- `my_string`은 영소문자로만 이루어져 있습니다.
- 1 ≤ `my_string`의 길이 ≤ 1,000
- `queries`의 원소는 [s, e]의 형태로 0 ≤ s ≤ e < `my_string`의 길이를 만족합니다.
- 1 ≤ `queries`의 길이 ≤ 1,000

## 입출력 예
| my_string | queries | result |
| :-- | :-- | :-- |
| "rermgorpsam" | [[2, 3], [0, 7], [5, 9], [6, 10]] | "programmers" |

## 입출력 예 설명
- 예제 1번의 `my_string`은 "rermgorpsam"이고 주어진 `queries`를 순서대로 처리하면 다음과 같습니다.

| queries | my_string |
| :-- | :-- |
| | "rermgorpsam"|
| [2, 3] | "remrgorpsam" |
| [0, 7] | "progrmersam" |
| [5, 9] | "prograsremm" |
| [6, 10] | "programmers" |

따라서 "programmers"를 return 합니다.

## 나의 풀이

### 첫 번째 풀이
```swift
func solution(_ my_string:String, _ queries:[[Int]]) -> String {
    
    var charArray = Array(my_string)
    
    for query in queries {
        charArray[query[0]...query[1]].reverse()
    }
    
    return String(charArray)
}
```

### 두 번째 풀이
```swift
func solution(_ my_string:String, _ queries:[[Int]]) -> String {
    String(queries.reduce(into: Array(my_string)) { charArray, query in
        charArray[query[0]...query[1]].reverse()
    })
}
```

## 배운점
- 이전에 공부했던 문자열을 가공하기 위해서는 Array를 사용하여 변환하는게 편리하다.
- `reverse()`
    - 배열 원본의 순서를 뒤집는다.
    - 추가적인 메모리 할당이 없어 반복문안에서 실행이되어도 성능면에서 매우 좋다.
- `reversed()`
    - 배열의 워본은 그대로 두고 뒤집힌 순서를 보여주는 새로운 뷰를 반환한다.
    - 루프마다 메모리에 새로운 임시 배열을 생성하므로 데이터가 많아질수록 성능 저하가 발생한다.
- 지금까지 고차함수를 사용하면 무조건 좋은줄 알았지만 아닌경우도 있다는것을 알았으며 체이닝이나 클로저 호출비용으로 인해 오히려 for문 보다 느릴수도 있다는 것을 알았다.
- 이전부터 생각없이 사용해왔던 reduce(into:)는 초기값을 inout 형태로 넘겨주어 메모리를 그대로 재활용 함으로 성능과 가독성면에서 좋다라는것을 알게되었다.
