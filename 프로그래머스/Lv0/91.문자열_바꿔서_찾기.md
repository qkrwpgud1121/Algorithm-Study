# 문자열 바꿔서 찾기

## 문제 설명
문자 "A"와 "B"로 이루어진 문자열 `myString`과 `pat`가 주어집니다. `myString`의 "A"를 "B"로, "B"를 "A"로 바꾼 문자열의 연속하는 부분 문자열 중 `pat`이 있으면 1을 아니면 0을 return 하는 solution 함수를 완성하세요.

## 제한사항
- 1 ≤ `myString`의 길이 ≤ 100
- 1 ≤ `pat`의 길이 ≤ 10
- `myString`과 `pat`는 문자 "A"와 "B"로만 이루어진 문자열입니다.

## 입출력 예
| myString | pat | result |
| :-- | :-- | :-- |
| "ABBAA" | "AABB" | 1 |
| "ABAB" | "ABAB" | 0 | 

## 입출력 예 설명
입출력 예 #1
- "ABBAA"에서 "A"와 "B"를 서로 바꾸면 "BAABB"입니다. 여기에는 부분문자열 "AABB"가 있기 때문에 1을 return 합니다.

입출력 예 #2
- "ABAB"에서 "A"와 "B"를 서로 바꾸면 "BABA"입니다. 여기에는 부분문자열 "BABA"가 없기 때문에 0을 return 합니다.

## 배운점
- `.contains`메서드는 자체적으로 Bool값을 반환하기때문에 불필요한 비교 연산자를 제거 하는것이 더 직관적이다.
- 첫번째 풀이에서는 `myString`전체를 치환하였지만 만약 `myString`이 매우 커질 경우 치환을 하기 위해 많은 메모리의 재할당이 발생한다는것을 알게 되었다.
- 해서 `myString`을 치환하는것이 아닌 비교적 작은 `pat`문자열을 치환하여 `myString`에서 탐색을 해도 결과가 같다는 것을 알게되었다.

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ myString:String, _ pat:String) -> Int {
    myString.map { $0 == "A" ? "B" : "A" }.joined().contains(pat) == true ? 1 : 0
}
```

### 두번째 풀이
```swift
func solution(_ myString:String, _ pat:String) -> Int {
    myString.contains(pat.map { $0 == "A" ? "B" : "A"}.joined()) ? 1 : 0
}
```
