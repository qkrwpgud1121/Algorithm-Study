# 공백으로 구분하기 1

## 문제 설명
단어가 공백 한 개로 구분되어 있는 문자열 `my_string`이 매개변수로 주어질 때, `my_string`에 나온 단어를 앞에서부터 순서대로 담은 문자열 배열을 return 하는 solution 함수를 작성해 주세요.

## 제한사항
- `my_string`은 영소문자와 공백으로만 이루어져 있습니다.
- 1 ≤ `my_string`의 길이 ≤ 1,000
- `my_string`의 맨 앞과 맨 뒤에 글자는 공백이 아닙니다.

## 입출력 예
| my_string | result |
| :-- | :-- |
| "i love you" | ["i", "love", "you"] |
| "programmers" | ["programmers"] |

## 입출력 예 설명
입출력 예 #1
예제 1번의 `my_string`은 "i love you"로 공백 한 칸으로 나누어진 단어들은 앞에서부터 순서대로 "i", "love", "you" 이므로 ["i", "love", "you"]를 return 합니다.

입출력 예 #2
예제 2번의 `my_string`은 "programmers"로 단어가 하나만 있습니다. 따라서 ["programmers"]를 return 합니다.

## 배운점
- `split(separator:)`는 원본 문자열의 메모리를 공유하는 [Substring]을 반환한다.
    - 해당 문제해결을 위해서는 `.map { String($0) }`을 사용해 명시적 타입 변환이 필요하다.
- `components(separatedBy:)`는 반환시 [String]을 메모리에 할당하여 새로 할당하여 반환한다.

- 만약 데이터의 조건이 1000이 아닌 1억 이상이 된다면 `components`를 사용하거나 `split`직후  바로`map`을 사용하면 1억 번 이상의 메모리 복사가 발생하여 크래시가 발생할 수 있다는것을 알게되었다.

- 따라서 `split`을 사용하여 [Substring] 상태 그대로 `filter`,`contains`등의 체이닝 연산을 하여 최소한의 데이터만 String으로 캐스팅 하는것이 최적화 하는 방식임을 알게되었다.

- 또한 `.lazy`를 사용하여 연산을 지연시키면 문자열을 찾는 순간 이후의 불필요한 전체 연산을 바로 종료하여 성능을 향상시킬 수 있다.

- 데이터 파이프라인의 연산 계급 - `지연 평가(Lazy Evaluation)`기반의 파이프라인은 두가지의 연산으로 나누어 동작한다.

    - `split`, `filter`, `lazy`등은 실제 데이터를 즉시 처리 하지 않고 실행 계획만 Stack 메모리에 저장한채로 대기하는 것을 **중간 연산 (Intermediate Operation)** Swift 에서는 **Lazy Sequence** 라고 한다.
    
    - `.first`, `.count`, `Array()`등은 파이프라인의 끝에서 Trigger 역할을 하며 이 코드가 호출되는 순간에 데이터가 파이프라인을 타고 실제 연산을 시작하는 것을 **최종 연산 (Terminal Operation / Trigger)** Swift 에서는 **Consuming** 라고 한다.
    
- 메모리 누수의 방지

    - 파이프라인 연산이 모두 끝이 나면 연산의 결과값(Substring)을 변수에 저장하거나 UI에 반영할때 반드시 String으로 다시 캐스팅 해주어야 한다.
    
    - Substring은 원본 문자열의 메모리를 참조하는 포인터 역할이므로 이를 String이라는 새로운 메모리 공간으로 복사하여 연결 고리를 끊어내지 않는다면 거대한 원본 데이터가 `자동 참조 계수(ARC)`에 의해 해제되지 못하고 메모리에 남아 누수가 발생한다는 것을 알게되었다.
    

### 예시
```swift
let firstLove = my_string
    .split(separator: " ")
    .lazy
    .filter { $0.contains("love")}
    .first
    .map { String($0) }
```

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ my_string:String) -> [String] {
    my_string.components(separatedBy: " ")
}
```

### 두번째 풀이
```swift
func solution(_ my_string:String) -> [String] {
    my_string.split(separator: " ").map { String($0) }
}
```
