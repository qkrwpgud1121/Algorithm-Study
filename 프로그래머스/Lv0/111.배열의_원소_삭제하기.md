# 배열의 원소 삭제하기

## 문제 설명
정수 배열 `arr`과 `delete_list`가 있습니다. `arr`의 원소 중 `delete_list`의 원소를 모두 삭제하고 남은 원소들은 기존의 `arr`에 있던 순서를 유지한 배열을 return 하는 solution 함수를 작성해 주세요.

## 제한사항
- 1 ≤ arr의 길이 ≤ 100
- 1 ≤ arr의 원소 ≤ 1,000
- arr의 원소는 모두 서로 다릅니다.
- 1 ≤ delete_list의 길이 ≤ 100
- 1 ≤ delete_list의 원소 ≤ 1,000
- delete_list의 원소는 모두 서로 다릅니다.

## 입출력 예

| arr | delete_list | result |
| :-- | :-- | :-- |
| [293, 1000, 395, 678, 94] | [94, 777, 104, 1000, 1, 12] | [293, 395, 678] |
| [110, 66, 439, 785, 1] | [377, 823, 119, 43] | [110, 66, 439, 785, 1] |

## 입출력 예 설명

입출력 예 #1
- 예제 1번의 `arr`의 원소 중 1000과 94가 `delete_list`에 있으므로 이 두 원소를 삭제한 [293, 395, 678]을 return 합니다.

입출력 예 #2
- 예제 2번의 `arr`의 원소 중 `delete_list`에 있는 원소는 없습니다. 따라서 `arr` 그대로인 [110, 66, 439, 785, 1]을 return 합니다.

## 배운점
- Array와 Set의 데이터 탐색 차이
    - Array (순차 탐색)
        - 특정 요소를 찾기 위해서는 배열의 맨 앞에서부터 하나씩 뒤로가며 찾아야 한다.
        - 찾는 요소가 뒤쪽에 있거나 없으면 배열의 길이만큼 처음 부터 끝까지 찾아야 하므로 시간의 복잡도가 높다.
        
    - Set (해시 테이블)
        - `Hash Function`을 거쳐 고유한 주소를 가진 `Hash Value`를 생성한다.
        - 생성된 `Hash Value`를 메모리의 `Hash Function`을 통해 계산된 고유 주소 인덱스에 메핑되는 `Bucket`에 저장한다.
        - 배열 처럼 순차적으로 찾지 않고 검색하고자 하는 값을 `Hash Function`에 대입하여 메모리 주소 인데스를 즉시 계산후 해당 `Bucket`으로 이동하여 탐색한다.
        - 데이터가 많아도 항상 **상수 시간** 만에 탐색이 완료된다.
        
        **해시 충돌 (Hash Collision)**
      
        - 서로 다른 데이터가 `Hash Function`을 거쳤을때 동일한 `Hash Value`를 가졌을때 하나의 `Bucket`을 두고 충돌하는 현상
        - **Chaining**
            - 충돌이 발생한 `Bucket`내부를 `Linked List` 구조로 연결하여 `Bucket` 내부의 데이터를 차례로 줄 세워 연결하는 방식이다.
            - Swift가 내부적으로 `Bucket`의 밀도가 높아질때 활용하는 방식이다.
        - **Open Addressing**
            - 계산된 `Bucket`의 주소에 이미 저장되어있는 데이터가 있다면 비어있는 `Bucket`을 찾을때까지 다음주소를 순차적으로 탐색하여 비어있는 `Bucket`에 데이터를 저장하는 방식이다.
        
        - 개발자가 `Set`,`Dictionary`와 같은 Swift 표준 라이브러리 자료구조를 사용할때는 Hash Collision을 방지하기 위해 직접 Chaining, Open Addressing을 할 필요가 없으며 Swift의 내부에서 자동으로 처리한다.
        
        - 하지만 만약 구조체(Struct)같은 모델을 Set에 넣으려고 하면 Hash Table의 index를 계산할 Key가 무엇인지 지정을 해주어야 한다.
        
        **Hashable 프로토콜 구현**
        ```swift
        func hash(into hasher: inout Hasher) {
            hasher.combine(id)
        }
        ```
        
- 첫번째 풀이에서 배열을 복사하는 연산, 삭제할 요소가 있는 배열의 반복, 반복문 내부에서의 순차적 요소 검색, 요소 삭제를 하며 요소들을 앞으로 한 칸씩 이동시키는 메모리 이동 작업에 대한 성능의 저하를 다시한번 깨닫게됨

## 나의 풀이

### 첫번째 풀이
```swift
func solution(_ arr: [Int], _ delete_list: [Int]) -> [Int] {
    
    var result = arr
    
    for i in delete_list {
        if result.contains(i) {
            result.remove(at: result.firstIndex(of: i)!)
        }
    }
    
    return result
}
```

### 두번째 풀이
```swift
func solution(_ arr: [Int], _ delete_list: [Int]) -> [Int] {
    
    let deleteSet = Set(delete_list)
    
    return arr.filter { !deleteSet.contains($0) }
}
```
